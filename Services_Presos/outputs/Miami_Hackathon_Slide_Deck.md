## Tool Mix Across All Projects

- **Claude** — 10 of 12 projects (main engine)
- **n8n** — 4 projects (workflow glue)
- **ChatGPT** — 3 projects (prompt design, secondary)
- **Gemini** — 1 project (LBO data extraction)
- **Azure OpenAI** — 2 projects (enterprise path)
- **Lovable** — 1 project (full-stack UI build)
- **Keye** — 1 project (winning specialist tool)

## Common Learnings Across Teams

1. Claude wins most head-to-heads — long prompts and Excel handling
2. n8n is the connector of choice for multi-step automation
3. Build almost always beat buy — one clean exception (Keye)
4. 1–2 week MVP is the right starting bar
5. Prompts are the product — iterate them, don't replace the model
6. Show reasoning — trust comes from transparency
7. Keep humans in the last mile
8. Data access is the real bottleneck, not model quality
9. Split AI (reading) from code (math) when numbers matter
10. Templates drive adoption — same shape every time

## Risks & Limits Called Out by Teams

- Context window limits on large documents
- LinkedIn and other gated sites blocked from scraping
- PitchBook MCP data patchy for smaller European companies
- AI still makes mistakes on complex Excel and multi-tranche debt
- GDPR and data sensitivity rules on candidate data
- Some outputs need "narrow" prompts — no one-size-fits-all yet

## Open Asks for Leadership

1. **Data access** — open LinkedIn, Affinity, PitchBook, and PortCo data to these tools safely
2. **Enterprise hosting** — get Lovable and similar tools inside the Permira tenant with SSO
3. **Gemini decision** — confirm whether Gemini fits the infra for LBO rollout
4. **Claude Cowork / Code access** — several teams want this for the next phase
5. **Light governance** — a small RAI / testing framework so tools can scale without risk
